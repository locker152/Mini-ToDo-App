# ​个人任务管理器（Mini ToDo App）
## 项目说明
本项目用来添加和管理使用者的任务
## 功能说明
1. 任务的添加
2. 任务的删除
3. 任务的状态更改
4. 任务的筛选（针对完成和未完成）
## 实现思路
### 任务数据的保存
通过变量newtask获取到从输入栏中输入的内容，然后再分别使用cell1  cell2  cell3来添加到使用newrow得到的新的一行元素的对应三个位置，从而实现了任务数据的保存
### 页面更新方法
每次添加完任务之后，都会重新执行一次filterapply函数，从而保证了页面更新后仍然可以筛选成功
### 添加，删除，状态切换功能的实现方法
##### 1.添加方法：
通过变量newtask获取到从输入栏中输入的内容，然后再分别使用cell1  cell2  cell3来添加到使用newrow得到的新的一行元素的对应三个位置，从而实现了任务的添加
#### 2.删除方法
给删除按钮绑定一个匿名函数，点击后可以直接remove掉新的一行的数据，从而实现了删除功能
#### 3.状态切换功能
在每一栏任务后面添加两个名为完成和未完成的按钮，在点击对应状态的按钮之后，对应的任务会发生相应的变化，如添加删除线和字体颜色变灰
#### 4.筛选功能的实现
通过使用filterapply这个函数来实现对筛选后的任务的显示或者是隐藏，通过bindfilter这个函数，使用事件监听来获取用户想要筛选的任务状态，再通过filterapply来进行实现
## 遇到的问题
#### 1.所有的状体按钮由于共用一个name，导致所有的任务状态只能选择一个，不能正常实现每一个任务的状态更改
#### 解决方案：
给每一个状态按钮添加一个动态的name，使用newid++来进行动态name的更新，再使用'&{newid}'把动态的name添加到input上

#### 2.选择一个筛选之后，不能在进行第二次的筛选了
#### 解决方案：
通过查询资料发现，添加任务后可能会找不到要绑定的元素，需要在jss的的末尾添加一个
```
window.onload = function() {
    bindfilter()    
    filterapply()   
}
```
函数才可以成功进行后续所有的筛选了
